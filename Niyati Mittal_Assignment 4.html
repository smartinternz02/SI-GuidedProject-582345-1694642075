Q1 = df['MonthlyIncome'].quantile(0.25)

# Define a threshold to identify outliers (e.g., 1.5 times IQR)
threshold = 1.5 * IQR

Performance Metrices

y_true: The true labels (ground truth) for the test data.

y_pred: The predicted labels from your classification model for the test data.

Accuracy: It measures the proportion of correctly classified instances among all instances. It's a good overall measure of model performance when classes are balanced. However, it can be misleading if classes are imbalanced.

Precision: Precision measures the accuracy of positive predictions. It's the ratio of correctly predicted positive instances to the total predicted positive instances. High precision indicates a low false positive rate.

Recall: Recall, also known as sensitivity or true positive rate, measures the ability of the model to identify all relevant instances. It's the ratio of correctly predicted positive instances to the total actual positive instances.

F1-Score: The F1-score is the harmonic mean of precision and recall. It provides a balance between precision and recall. It is useful when you want to find an optimal balance between minimizing false positives and false negatives.

Confusion Matrix: A confusion matrix is a table that summarizes the model's performance by comparing actual and predicted classifications. It contains four values: true positives (TP), true negatives (TN), false positives (FP), and false negatives (FN). It helps you understand where your model is making errors.

Classification Report: The classification report provides a comprehensive summary of precision, recall, F1-score, and support for each class. It's particularly useful when dealing with multi-class classification problems.
